# VidenX 后端项目技术文档

## 项目概述

VidenX 是一个 AI 视频生成平台的后端服务，基于 NestJS 框架构建，支持多平台视频生成（Sora、Polo、Dayangyu 等），集成了飞书机器人、用户积分系统、云存储等功能。

**项目路径**: `E:\projects\works\aigc-videnx\backend`

---

## 技术栈概览

### 核心框架
- **NestJS v11** - 基于 TypeScript 的 Node.js 后端框架
- **TypeORM** - 数据库 ORM 框架，支持 MySQL
- **Express** - HTTP 服务器底层框架

### 语言与构建
- **TypeScript 5.7** - 开发语言
- **Node.js** - 运行时环境
- **ts-node** - 开发环境运行
- **PM2** - 生产环境进程管理
- **Nginx** - 反向代理（部署配置）

### 依赖管理
- **pnpm** - 包管理器

### 数据库
- **MySQL 8.0** - 主数据库
- **MongoDB** - 辅助数据存储（部分功能）

### 云服务与存储
- **阿里云 OSS** - 主要对象存储
- **Cloudflare R2** - 备用对象存储
- **Google Drive** - 视频文件存储

### 第三方集成
- **飞书** - 消息通知、机器人、日志记录
- **OpenAI Sora** - 视频生成平台
- **LangChain** - AI 能力封装
- **Playwright** - 浏览器自动化

### 网络与代理
- **HTTP/2** - 网络传输协议
- **Socks5 Proxy** - 代理支持
- **自定义 Spider 模块** - 网络爬虫框架

## 核心业务模块详解

### 1. 视频生成任务流程

系统采用 **多阶段任务执行器** 模式处理视频生成：

```
任务创建 → 任务组生成 → 任务分发 → 视频创建 → 状态轮询 → 完成检查
   ↓           ↓            ↓           ↓          ↓          ↓
 Task     TaskGroup    Dispatch    Create    Polling   Completion
```

#### 任务执行器（Runner）

| 执行器 | 功能 | 触发间隔 |
|--------|------|----------|
| `GroupVideoRunner` | 待处理任务 → 任务组 | 15秒 |
| `DispatchVideoRunner` | 任务组 → 视频产物 | 5秒 |
| `CreateSoraVideoRunner` | 创建平台任务 | 15秒 |
| `PollingSoraVideoRunner` | 轮询平台状态 | 20秒 |
| `TaskCompletionRunner` | 完成检查与评分 | 10秒 |

### 2. 用户积分系统

- **用户角色**: `ADMIN` / `OPERATOR` / `CUSTOMER`
- **用户状态**: `ACTIVE` / `DISABLED`
- **积分操作类型**:
  - `TASK_CONSUME` - 任务消耗
  - `TASK_REFUND` - 任务退款
  - `ADMIN_ADJUST` - 管理员调整

### 3. 第三方平台集成

#### Sora 视频生成
- `SoraWebApiService` - Sora 官方 Web API 封装
- 支持视频创建、上传、分享、状态查询
- 使用 HTTP/2 + Socks5 代理访问

#### Polo / Dayangyu
- `PoloWebApiService` - Polo 平台 API
- `DayangyuWebApiService` - 大洋屿平台 API

### 4. 飞书集成

- 消息通知推送
- 日志记录到飞书文档
- 机器人告警（普通/重要）
- Bitable（电子表格）数据同步

### 5. AI 能力

#### AiRouter - AI 路由器
统一接入不同 AI 平台（OpenRouter、AI12 等）

#### VideoAnalysisAgent - 视频分析 Agent
- 基于 LangChain 框架
- 支持视频 URL 或 Base64 输入
- 输出结构化分析结果（标题、分镜脚本）

## 开发规范

### 编码规范

#### 1. 依赖注入
**禁止**使用构造函数注入，**必须**使用 `@Inject()` 装饰器：

```typescript
@Injectable()
export class MyService extends BaseService {
  @Inject()
  private readonly otherService: OtherService;
}
```

#### 2. 服务继承
- 所有业务服务 **建议**继承 `BaseService`
- 利用 `this.config('key')` 获取配置
- 所有控制器 **必须**继承 `BaseController`

#### 3. 基类使用

**BaseService**:
```typescript
export default class BaseService {
  @Inject()
  protected readonly configService: ConfigService;

  protected config<T extends keyof IConfig>(cfgKey: T): IConfig[T] {
    return this.configService.get<IConfig[T]>(cfgKey);
  }
}
```

**BaseController**:
```typescript
export default class BaseController {
  @Inject()
  protected readonly forge: TypeOrmForge;

  public baseSuccess<T>(data?: T) {
    return { success: true, data };
  }

  public baseError(errMsg?: string, errCode: number = -1) {
    return { success: false, errMsg, errCode };
  }
}
```

### 数据库规范

#### Entity 定义
- 每个表单独一个文件，命名为 `xxx.entity.ts`
- 枚举类型定义在实体文件内部
- **必须**为每个字段添加中文备注
- **禁止**生成外键关联（仅保留关联 ID）
- 所有表包含 `id` (UUID)、`createdAt`、`updatedAt`
- 索引统一在类装饰器层级定义

```typescript
@Entity('tasks')
@Index(['status'])
@Index(['operatorUserId'])
export class Task extends Base {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({ comment: '任务名称' })
  taskName: string;
}
```

### API 开发规范

#### 路由注册顺序
```
1. 模型结构 (entity)
2. 仓库聚合 (model/service)
3. Handler 组合响应 (controller)
4. 路由注册
5. 验证交付
```

#### 目录导向
- 通用组件与模板: `src/providers/template/`
- 数据模型定义: `src/entity/`
- 模型处理: `src/model/`
- 业务模块: `src/module/`

## 代码示例

### 创建新 API 接口

1. **定义 Entity** (`src/entity/example.entity.ts`):
```typescript
import { Entity, Column, PrimaryGeneratedColumn, Index } from 'typeorm';
import { Base } from '../providers/template/base.entity';

@Entity('examples')
@Index(['status'])
export class Example extends Base {
  @PrimaryGeneratedColumn('uuid')
  id: string;

  @Column({ comment: '名称' })
  name: string;

  @Column({ comment: '状态' })
  status: string;
}
```

2. **创建 Model Service** (`src/model/example.service.ts`):
```typescript
import Injectable from '@/src/providers/template/base.service';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository } from 'typeorm';
import { Example } from '@/src/entity/example.entity';

@Injectable()
export class ExampleService extends BaseService {
  @InjectRepository(Example)
  private readonly exampleRepo: Repository<Example>;

  async findAll() {
    return this.exampleRepo.find();
  }
}
```

3. **创建 Controller** (`src/module/api/controller/public/example.controller.ts`):
```typescript
import { Controller, Inject, Get } from '@nestjs/common';
import BaseController from '@/src/providers/template/base.controller';
import { ExampleService } from '@/src/model/example.service';
import { ApiTags, ApiOperation } from '@nestjs/swagger';

@Controller('example')
@ApiTags('Example')
export class ExampleController extends BaseController {
  @Inject()
  private readonly exampleService: ExampleService;

  @Get('list')
  @ApiOperation({ summary: '获取列表' })
  async list() {
    const data = await this.exampleService.findAll();
    return this.baseSuccess(data);
  }
}
```

4. **注册到 Module**:
```typescript
// 在对应的 module.ts 中添加
@Module({
  controllers: [ExampleController],
  providers: [ExampleService],
})
export class PublicApiModule {}
```

## 常见问题

### 1. 如何添加新的第三方平台？

参考 `src/module/common/api/sora.web.api.ts`，创建新的 Web API 服务类，然后在 `src/module/task/provider/video.provider.ts` 中注册。

### 2. 如何配置新的 AI 平台？

在 `src/config/ai.ts` 中添加平台配置，然后实现对应的 Provider（参考 `src/providers/ai/platforms/ai12.platform.ts`）。

### 3. 如何添加定时任务？

使用 `@nestjs/schedule` 装饰器，或在 `TaskService` 中注册新的 Runner。

### 4. 如何调试？

- 使用 `pnpm run start:dev` 启动热重载模式
- 在 VS Code 中设置断点调试
- 查看飞书日志机器人获取运行时日志
